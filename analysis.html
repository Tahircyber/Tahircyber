Welcome to the Analysis Page

This page is dedicated to providing in-depth analysis and research on privacy and security issues related to smart devices. I explore various topics, including vulnerability assessments, penetration testing, and forensic analysis, to shed light on potential risks and help users make informed decisions. Completing this analysis should show you how easy it can be for hackers to gain unauthorised access and is also provides an opportunity to show my skills.

Vulnerability Assessments

Vulnerability assessments involve identifying and documenting vulnerabilities in systems, networks, or applications. When I used tools like Nmap to scan smart devices connected to my network, I was essentially conducting a form of vulnerability assessment. Nmap can identify open ports, services running on those ports, and potentially vulnerable versions of those services.

Evidence

The image above shows me running the command 'ifconfig' on my Virtual Kali Linux machine, which is deployed using VirtualBox on my PC connected to my home network. We can see that this has been done correctly as we get an IP address beginning with 192.168, which usually refers to the IP addresses within a home network. Using this information, we can run various commands such as Nmap to scan for devices within my network.

As you can see, using the IP address of the Kali Linux machine, we can run an Nmap scan, which is short for Network Map on the entire network using the '/24' at the end of the IP address to scan all IP addresses within 24 bits and look for open ports on running devices.

From the two images above that display the output of the Nmap scan, we can see the running devices on my home network. For example, portrayed in the images you can see the SkyRouter open ports, a smart TV named DIXONS-JVC-TV, and two other devices which can be identified as iPhones even though it says unknown due to the simple fact that the iPhone-sync service is open on both. Identifying open ports is a great tool, as with it we can see any potential security flaws within the network via delving deeper into the ports that appear to be open and performing research on websites such as the CVE (Common Vulnerabilities and Exposures) webpage to determine whether or not the ports open are vulnerable to certain attacks. In my case, none of these open ports served an issue.

Penetration Testing

Penetration testing, or pen testing, involves simulating real-world attacks to identify security weaknesses in systems or applications. When I attempted to hack an emulated Android device using AndroRAT, I was essentially conducting a form of penetration testing. Penetration testing aims to exploit vulnerabilities to determine the impact they could have on a system's security.

Evidence

To implement a form of penetration testing, I used the AndroRat GitHub repository to present how easy it is for hackers to completely take over an Android phone using a link that can be sent to the user where if the user clicks on the link and doesn't pay much attention to the popups that appear before clicking install which a lot of people do. The image above just shows step one of the process which is cloning the git repository into my virtual machine and installing the prerequisites.

The image above shows the build process of the APK file. We use the -o command to output the file name into 'nothacking.apk' and the -i command to specify the IP address which in this case is the IP of the Kali Linux machine. The image shows how the command successfully created the file and we can see the file when we do the 'ls -lt' command to list the files in the current directory.

This image which can be seen above shows how a listener was hosted on port 4444 which was stated before.

Now that we have the listener set up, it's essential to make sure that we have an HTTP server running in order to allow us to host the file over onto the internet or local network. The start apache2.service command runs this and we can check that this has been run correctly by replacing the start with status. The image above shows this.

The final step to setting up AndroRat correctly is by copying the APK file that we created into the web application server directory which we can then send over to our target.

Forensic Analysis

Forensic analysis involves investigating and analyzing digital evidence to understand the cause of security incidents or to reconstruct events. When I used Wireshark to capture and analyse network traffic generated by Nmap scans, I was performing a form of forensic analysis. Wireshark allows me to inspect network packets, which can help me understand how devices communicate on the network and identify potential security issues.

Evidence

Wireshark is an extremely effective tool used to capture the traffic within a network and identify any security vulnerabilities within the network. The image above shows regular traffic within my network, showing nothing out of the ordinary.

To display an example of what the traffic would look like to raise cause for concern on Wireshark, I ran an Nmap scan on the network and used Wireshark to capture the traffic. We can see from the image above that this shows up highlighted red as it could be a potential for danger.
